# 软件评测指标

## 软件功能性指标（60分）

| 评分指标 | 子评分项 | 评分标准 |
| --- | --- | --- |
| 软件功能性指标(60分) | 稳定性(10分) | 1.场景鲁棒性:给定的用例和场景下运行正常(6分)<br>2.异常重建能力(4分)<br>●崩溃后可以自动重建，运行中容器不受影响(2分)<br>●重建过程确保文件数据完整性，包括文件元数据，去重索引数据(2分) |
|  | 完整性(30分) | 1.支持数据去重(10分)<br>2.支持按需加载(10分)<br>3.兼容OCI镜像分层逻辑(10分) |
|  | 先进性(20分) | 1.数据去重(10分):<br>●支持块粒度数据去重(3分)<br>●不破坏文件元数据(3分)<br>●支持内存去重(4分)<br>2.性能优化(5分):<br>●按需加载支持预取(3分)<br>●减少内核和用户态切换(2分)<br>3.兼容性(5分):<br>●原生支持OCIv1镜像格式 |

| 评分指标 | 子评分项 | 评分标准 |
| --- | --- | --- |

|  | 资源消耗(5分) | 1.CPU消耗(3分):idle状态CPU为0%，压力测试CPU峰值不超过20%<br>2.内存消耗(2分):常驻内存不超过1G |
|  | 测试(10分) | 1.单元测试(5分):覆盖去重逻辑、数据管理等核心模块功能验证<br>2.集成测试(3分):全链路镜像拉取-加载-容器启动流程<br>3.压力测试(2分):10+节点并发测试启动压力 |
|  | 开发设计文档(15分) | 1.详细设计方案(12分):<br>●架构图:展示数据去重与按需加载模块的交互关系及层级设计(4分)<br>●流程图:刻画容器冷启动全流程，包含数据去重、按需加载时序逻辑(4分)<br>●方案选型:对比分析Fuse/EROFS/Composefs方案优劣，含性能与兼容性论证(4分)<br>2.视频演示(3分):演示作品所有功能的运行效果 |
